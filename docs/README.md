## 미션 흐름에 따른 문제 이해

1. **로또 구매 금액 입력**
    1. 입력 값은 숫자여야 한다
    2. 입력 값은 1000으로 나누어떨어져야 한다
    3. 위 사항에 부합하지 않는 경우 예외 발생
        - 에러 문구는 “[ERROR]”로 시작되어야 한다

          → ex) [ERROR] 로또 번호는 1부터 45 사이의 숫자여야 합니다.


2. **로또 번호 출력**
    1. 구매 수량 출력
    2. 로또 번호 구매 수량만큼 출력
        - 이때, 로또 번호는 랜덤으로 주어진다
            - 모든 번호는 1 ~ 45 사이의 값으로 이루어져야 한다
            - 하나의 로또에는 중복 값이 있어선 안 된다
            - 로또 번호는 오름차순으로 정렬해 보여준다


3. **당첨 번호 입력**
    1. 번호는 쉼표로 구분해 6개를 입력받는다
    2. 입력된 당첨 번호가 유효한 값인지 확인한다
        - 쉼표로 split() 하여 6개의 값인지 확인
        - 모든 값이 숫자인지 확인
        - 모든 값이 1 ~ 45 범위에 충족하는지 확인
        - 중복되는 값이 없는지 확인
    3. 유효한 값이 아닐 시 예외 발생
        - 에러 문구는 “[ERROR]”로 시작되어야 한다

          → ex) [ERROR] 로또 번호는 1부터 45 사이의 숫자여야 합니다.


4. **보너스 번호 입력**
    1. 1 ~ 45 사이의 값을 하나의 번호를 입력받는다
    2. 입력된 보너스 번호가 유효한 값인지 확인한다
        - 값이 숫자인지 확인
        - 1 ~ 45 사이의 값인지 확인
        - 당첨 번호에서 중복되는 값이 없는지 확인
    3. 유효한 값이 아닐 시 예외 발생
        - 에러 문구는 “[ERROR]”로 시작되어야 한다

          → ex) [ERROR] 로또 번호는 1부터 45 사이의 숫자여야 합니다.


5. **당첨 통계 계산**
    1. 당첨 내역 개수 계산
    2. 총 수익률 계산
        - 구입 금액만큼의 소득이 있을 시 100.0%로 계산됨
        - 8000원을 구입하고 5000원을 번 경우

          → 계산 예시 : (5000 / 8000) * 100 = 62.5%

        - 수익률은 소수점 둘째 자리에서 반올림한다

          → ex. 100.0%, 51.5%, 1,000,000.0%


6. **당첨 통계 출력**
    1. 당첨 내역을 출력한다
    2. 총 수익률을 출력한다

## 기능에 따른 분류
### LottoStore

> 로또 가게로, 미션의 전체적인 흐름을 관리하는 클래스

- [x]  로또 구매
- [x]  로또 번호 알려주기 (출력)
- [x]  당첨 번호 입력
- [x]  보너스 번호 입력
- [x]  당첨 통계 보여주기 (출력)

### PurchaseAmount

> 구매 금액으로, 사용자가 입력한 로또 구매 금액을 저장하고, 유효성을 검증하는 클래스

- [x]  입력 값 유효성 검증
    - [x]  숫자인지 확인
    - [x]  0을 초과하는지 확인
    - [x]  1000으로 나누어떨어지는지 확인
    - [x]  유효성 검증 실패 시 오류 메시지 출력 (IllegalArgumentException 발생시키기)
        - [x]  입력 값이 없거나 숫자가 아닌 경우

           : [ERROR] 금액을 투입해 주셔야 합니다.

        - [x]  0 이하의 값을 입력한 경우

           : [ERROR] 0을 초과한 금액을 투입해 주셔야 합니다.

        - [x]  1000으로 나누어떨어지지 않는 경우

           : [ERROR] 입력 금액은 1,000원 단위여야 합니다.

- [x]  구매 금액 저장
- [x]  구매 금액에 따른 구매 로또 개수 계산
- [x]  구매 로또 개수 저장

### PurchaseLottos

> 사용자가 구매한 로또 번호를 자동으로 생성하고 저장해 주는 클래스

- [x]  LottoMachine 클래스를 통해 구매 수량만큼의 로또 생성
- [x]  생성한 로또 저장

### Lotto

> 입력받은 로또 당첨 번호를 저장하고, 유효성을 검증하는 클래스

- [x]  입력 값 유효성 검증
    - [x]  쉼표로 구분된 값이 6개인지 확인
    - [x]  모든 값이 숫자인지 확인
    - [x]  모든 값이 1 ~ 45 사이의 값들인지 확인
    - [x]  중복되는 값이 없는지 확인
    - [x]  유효성 검증 실패 시 오류 메시지 출력 (IllegalArgumentException 발생시키기)
        - [x]  입력 값이 없거나 값이 6개가 아닌 경우

           : [ERROR] 6개의 값을 쉼표로 구분해 작성해 주셔야 합니다.

        - [x]  숫자가 아닌 값이 있는 경우

           : [ERROR] 1 ~ 45 사이의 값만 입력해 주셔야 합니다.

        - [x]  1 ~ 45 사이 외의 값이 있는 경우

           : [ERROR] 1 ~ 45 사이의 값만 입력해 주셔야 합니다.

        - [x]  중복되는 값이 있는 경우

           : [ERROR] 서로 다른 숫자들을 입력해 주셔야 합니다.

- [x]  당첨 번호 저장
- [x]  당첨 입력 값을 List<Integer> 형태로 변환

### BonusNumber

> 입력받은 보너스 번호를 저장하고, 유효성을 검증하는 클래스

- [x]  입력 값 유효성 검증
    - [x]  값이 존재하는지 확인
    - [x]  구분 문자가 있는지 확인 (스페이스 or 쉼표)
    - [x]  숫자인지 확인
    - [x]  1 ~ 45 사이의 값인지 확인
    - [x]  당첨 번호에 없는 값인지 확인
    - [x]  유효성 검증 실패 시 오류 메시지 출력 (IllegalArgumentException 발생시키기)
        - [x]  값이 없거나 숫자가 아닌 경우

           : [ERROR] 1 ~ 45 사이의 값을 입력해 주셔야 합니다.
       
        - [x]  구분 문자로 값이 하나가 아닌 경우

           : [ERROR] 1 ~ 45 사이의 값을 하나만 입력해 주셔야 합니다.

        - [x]  1 ~ 45 사이의 값이 아닌 경우

           : [ERROR] 1 ~ 45 사이의 값을 입력해 주셔야 합니다.

        - [x]  당첨 번호에 있는 번호인 경우

            [ERROR] 당첨 번호에 없는 번호를 입력해 주셔야 합니다.

- [x]  보너스 번호 저장

### WinningStatisticsCalculator

> 당첨 통계 계산기로, 당첨 내역 개수와 수익률에 대해 계산하고, 저장하는 클래스

- [x]  당첨 내역 초기화
- [x]  당첨 번호와 로또 비교
    - [x]  당첨 번호와 로또 번호의 공통 값 개수를 센다
    - [x]  보너스 번호와 일치하는 값이 있는지 확인한다 
- [x]  당첨 내역 개수 계산
    - [x]  1등 ~ 5등 중 당첨 개수를 센다
- [x]  당첨 내역 값 저장
- [x]  당첨 금액 저장
- [x]  총 수익률 계산
    - [x]  수식 계산 : ((당첨 금액) / (구매 금액)) * 100
    - [x]  수익률 소수점 둘째 자리에서 반올림
- [x]  총 수익률 저장

### LottoMachine

> 조건에 부합하는 로또 번호를 랜덤으로 생성해 주는 클래스

- [x]  조건에 부합하는 로또 번호 생성
    - [x]  1 ~ 45 사이의 값으로 이루어져야 한다
    - [x]  중복 값이 존재해선 안 된다
- [x]  번호 오름차순으로 정렬

### InputView

> 사용자가 입력해야 하는 기능을 모아둔 클래스

- [x]  로또 구매 금액 입력
- [x]  로또 당첨 번호 입력
- [x]  로또 보너스 번호 입력

### OutputView

> 사용자에게 출력되는 기능들을 모아둔 클래스

- [x]  구매한 로또 번호 출력
- [x]  당첨 통계 출력
    - [x]  당첨 내역 출력
    - [x]  총 수익률 출력

## 예외 사항
> 예외 사항 발생 시 IllegalArgumentException를 발생시키고, "[ERROR]"로 시작하는 에러 메시지를 출력 후 종료한다.

### 로또 구매 금액 입력

1. 아무 값도 입력되지 않거나 띄어쓰기만 있는 경우
    - 오류 메시지 : [ERROR] 금액을 투입해 주셔야 합니다.

2. 1,000으로 나누어떨어지지 않는 경우
    - 오류 메시지 : [ERROR] 입력 금액은 1,000원 단위여야 합니다.

3. 0을 입력한 경우
    - 오류 메시지 : [ERROR] 0원을 초과한 금액을 투입해 주셔야 합니다.

### 당첨 번호 입력

1. 입력 값이 없거나 값이 6개가 아닌 경우
    - 오류 메시지 : [ERROR] 6개의 값을 쉼표로 구분해 작성해 주셔야 합니다.
2. 숫자가 아닌 값이 있는 경우
    - 오류 메시지 : [ERROR] 45 사이의 값만 입력해 주셔야 합니다.
3. 1 ~ 45 사이 외의 값이 있는 경우
    - 오류 메시지 : [ERROR] 45 사이의 값만 입력해 주셔야 합니다.
4. 중복되는 값이 있는 경우
    - 오류 메시지 : [ERROR] 서로 다른 숫자들을 입력해 주셔야 합니다.

### 보너스 번호 입력

1. 값이 없거나 숫자가 아닌 경우
    - 오류 메시지 : [ERROR] 45 사이의 값을 입력해 주셔야 합니다.
2. 구분 문자로 값이 하나가 아닌 경우
    - 오류 메시지 : [ERROR] 45 사이의 값을 하나만 입력해 주셔야 합니다.
3. 1 ~ 45 사이의 값이 아닌 경우
    - 오류 메시지 : [ERROR] 45 사이의 값을 입력해 주셔야 합니다.
4. 당첨 번호에 있는 번호인 경우
    - 오류 메시지 : [ERROR] 당첨 번호에 없는 번호를 입력해 주셔야 합니다.
